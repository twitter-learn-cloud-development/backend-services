# 项目开发规范

## 文档自动维护规则

### 1. 规划与实现跟踪 (docs/PROJECT_PROGRESS.md)
- 每次完成功能模块后，自动更新对应任务状态（⬜→✅）
- 新增阶段时沿用现有的表格格式
- 如果实现方案与原计划有偏差，在任务行备注偏差原因

### 2. 接口文档 (docs/API_REFERENCE.md)
- 每次新增或修改接口后，自动追加到对应分类下
- 严格沿用现有格式：请求方法、路径、请求体、成功响应
- 包含路径参数表格和 JSON 示例

### 3. 问题追踪 (docs/ISSUES.md)
- 当功能验证失败（编译错误、测试不通过、运行时异常）时，自动追加记录
- 沿用现有的表格格式：问题、原因、解决
- 问题解决后在同一条目中补充解决方案
